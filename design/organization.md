# 公司组织架构

游戏开始时选择公司架构，影响**决策机制**、**手牌上限**、**策略卡协同效果**。

---

## 架构一览

### 1. 扁平化创业团队（Flat Startup）

**适合**：2-3 人游戏，快速迭代风格

```
创始人
├── 工程
├── 产品
└── 增长
```

**规则效果**
- 所有角色共享一个「团队行动池」，每回合可联合使用 3 张牌（而非各自 1 张）
- 决策无需投票，创始人单独决定
- 手牌上限 +2（每人最多持有 7 张）
- 技术债务增长速度 +10%（人少事多）
- 若团队士气 > 80%，每回合额外抽 1 张策略卡

**适合路线**：开源路线（快速迭代，社区驱动）

---

### 2. 功能型组织（Functional Org）

**适合**：3-5 人游戏，部门分明

```
CEO
├── 技术部（CTO）
├── 市场部（CMO）
└── 销售部（VP Sales）
```

**规则效果**
- 每个部门独立手牌，互不干扰
- 策略卡按部门分类，跨部门使用需支付「协调成本」（效果 -10%）
- 每回合顺序：CEO → 技术 → 市场 → 销售
- 商业化策略卡效果 +15%

**适合路线**：商业化路线（分工明确，销售驱动）

---

### 3. 矩阵式组织（Matrix Org）

**适合**：4-6 人游戏，复杂博弈

```
CEO
├── 职能线（CTO / CMO / VP Sales）
└── 产品线（产品 A / 产品 B）
```

**规则效果**
- 每张策略卡同时归属「职能」和「产品」两个维度
- 双维度匹配时，卡牌效果 +25%（协同加成）
- 每回合存在「资源争夺」阶段：两条产品线竞争同一张关键资源卡
- 士气波动更大（矩阵汇报关系复杂）：每回合士气随机 ±5%

**适合路线**：双轨平衡（同时维护开源产品线和商业产品线）

---

### 4. 公会制 / 小队制（Squad / Guild）

**适合**：Spotify 模型爱好者，4-6 人

```
Chapter Lead（技术）  Chapter Lead（设计）
      │                      │
   Squad A ────────────── Squad B
（开源社区）             （企业客户）
```

**规则效果**
- 玩家分为 2 支小队，各自推进不同目标（开源指数 vs 商业化指数）
- 每 3 回合举行一次「Chapter Review」：落后小队可借用 1 张领先小队的策略卡
- 允许小队内部交换手牌
- 整体团队士气取两队平均值

**适合路线**：双轨平衡或对抗模式

---

### 5. 开源基金会模型（Foundation Model）

**适合**：开源路线深度玩法，3-5 人

```
技术指导委员会（TSC）
├── 核心维护者
├── 贡献者代表
└── 最终用户代表（企业会员）
```

**规则效果**
- 所有重大策略卡需要「TSC 投票」（多数通过）才能生效
- 引入「企业会员」机制：企业赞助 = 现金流来源（替代传统销售）
- 社区影响力超过 60% 时，每回合自动吸引 1 名新贡献者
- 无法打出纯商业销售类卡牌，但可打出「企业服务支持」变体

**适合路线**：开源路线（Linux Foundation / Apache 风格）

---

### 6. 控股集团 / 商业版本并行（Dual-Track Corp）

**适合**：Open-Core 策略，4-6 人

```
母公司（商业实体）
├── 开源项目（社区维护）
└── 商业产品（企业版本）
```

**规则效果**
- 游戏内存在两套独立的策略卡堆：「社区牌堆」和「企业牌堆」
- 每回合各抽 1 张，可选择打出哪一张
- 若开源指数和商业化指数差距 < 20%，进入「双轨加速」状态：效果全部 +10%
- 若差距 > 50%，触发「路线撕裂」事件

**适合路线**：双轨平衡（HashiCorp / Elastic 风格）

---

## 架构对比

| 架构 | 决策速度 | 协同难度 | 适合路线 | 特色机制 |
|------|---------|---------|---------|---------|
| 扁平创业 | 极快 | 低 | 开源 | 联合行动池 |
| 功能型 | 快 | 中 | 商业化 | 部门专属加成 |
| 矩阵式 | 慢 | 高 | 双轨 | 双维度协同 |
| 公会制 | 中 | 中 | 双轨/对抗 | 小队竞争与协作 |
| 基金会 | 很慢 | 很高 | 开源 | 投票机制 |
| 双轨集团 | 中 | 中 | 双轨 | 双牌堆系统 |
